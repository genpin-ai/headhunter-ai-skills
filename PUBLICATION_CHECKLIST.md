# Publication Checklist

Use this checklist before publishing the public preview.

## Local Asset Check

- [ ] `README.md` states that the repository is a public preview and not open source.
- [ ] `LICENSE` uses all-rights-reserved language.
- [ ] `NOTICE.md` warns against sharing sensitive client or candidate data.
- [ ] Every skill folder contains `SKILL.public.md`, not the internal `SKILL.md`.
- [ ] Public examples are fictional or fully sanitized.
- [ ] No private names, phone numbers, emails, resumes, salary data, interview notes, offer details, or client documents are included.

## GitHub Repository

- [ ] Create a new public repository: `genpin-ai/headhunter-ai-skills`.
- [ ] Push only the public-preview package.
- [ ] Disable Issues.
- [ ] Enable Discussions only if you are ready to moderate public questions.
- [ ] Add a repository description: `Public preview of AI-assisted recruiting delivery skills. Not open source.`
- [ ] Add topics such as `ai-skills`, `recruiting`, `headhunter`, `workflow`, `public-preview`.
- [ ] Create first release: `v0.1-public-preview`.
- [ ] Do not upload private packages or internal artifacts to the release.

## Website

- [ ] Publish `genpin-personal-site/headhunter-ai-skills.html`.
- [ ] Confirm `https://declan-zhao.com/headhunter-ai-skills` loads correctly.
- [ ] Confirm the CTA points to the desired contact path.
- [ ] Update the GitHub button after the public repo exists.
- [ ] Submit or refresh sitemap if needed.

## Promotion

- [ ] First public note: explain why recruiting teams need workflow skills rather than one-off prompts.
- [ ] Second public note: show the 9-skill workflow map.
- [ ] Third public note: show one sanitized example fragment.
- [ ] Private outreach: send the website page, not the raw GitHub repo, to potential clients.

## Quality Gate

Do not publish if any public file exposes private client, candidate, salary, interview, offer, or internal method details.
