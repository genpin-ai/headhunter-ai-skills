# Resume Recommendation Report / 简历报告

Public preview version: v0.1.

This is a public preview, not the complete internal skill.

## Purpose

Generate a consultant-ready candidate recommendation report from a resume and optional JD, client context, communication notes, or interview material.

## When To Use

- A candidate is ready to be recommended to a client.
- A resume needs to be converted into a structured recommendation report.
- A team needs consistent candidate facts, advantages, and assessment boundaries.

## Typical Inputs

- resume file or text
- candidate structured information
- target role or JD
- communication notes or consultant assessment
- confirmed missing-field decisions

## Expected Outputs

- personal information
- education background
- core advantages
- work experience
- project experience
- AI-assisted assessment
- missing field and confirmation notes

## Boundaries

- Facts must be extracted, not invented.
- Missing required fields should be requested or explicitly confirmed as placeholders.
- AI assessment must be evidence-bound and reviewed before external use.
- Reports should remain editable and consultant-controllable.

The full version may include structured extraction, confirmation gates, Word output, layout rules, and system export formats under a private license.
